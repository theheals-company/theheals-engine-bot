## 심판 계층 정의 (마스터 도면 §2 ⑤, 2026-07-11 확정) ⚠️ 신설

- **CI심판**: 발주 생애주기의 **1차** 검증 단계. **GitHub Actions 기반 자동 검증**
  (린트·테스트·빌드)을 가리킨다. 판정자는 사람이 아닌 CI 파이프라인(ci.yml 3잡:
  Lint/Test/Build).
- **감리**: 발주 생애주기의 **2차** 검증 단계. **교차모델 리뷰**(cross_review, 현재
  리뷰어 GPT-5.5)를 가리킨다. CI심판 다음, 승인대기 이전 단계.
- ⚠️ **금지 해석**: 두 단계를 동일시하거나 서로 대체 가능하다고 서술 금지. CI심판과
  감리는 별개이며 병합 대상이 아니다.
- ⚠️ **현재 구현 상태 메모 (진단, 2026-07-11 — 기록용, 아직 수정 안 됨)**: 관제봇
  `on_message` 파이프라인은 현재 "CI심판" 상태를 전이·게시만 하고, 그 사이에 실제
  GitHub Actions 호출/판정 로직이 없다(빈 전이). 실제로 실행되는 자동판정은
  `cross_review` 하나뿐이며 이는 (올바르게) "감리" 라벨로 게시된다. 즉 cross_review가
  CI심판 자리를 대신 채운 적은 없음 — CI심판은 애초에 기능 없는 라벨이었다. 향후
  PR·커밋이 연동된 코딩형 ORD에 한해 GitHub Checks API 조회를 CI심판 단계에 연결하는
  구현이 필요(미착수, 이 문서는 진단만 기록).

## 기술 용어 (⚠️ 금지 해석 포함)

- **IOBT / IoBT = Inside-Out Body Tracking**: Quest 등 헤드셋이 외부 센서 없이
  내장 카메라만으로 손·몸 관절을 추적하는 기술.
  ⚠️ **금지 해석: "Internet of Battlefield Things"(전술 사물인터넷)가 절대 아님.
  군사/국방/드론/전장 맥락으로 해석 금지.** (2026-07-11 실제 오인 사고 발생)
  참고 기준 데이터: 손가락 관절각 평균 오차 약 9.6°, 폐색(occlusion) 시 5~15°
  추가 오차 및 손가락 동결(freeze) 현상.
- **Virtual Tension (가상 장력)**: 시술자의 손가락 관절각 등 키네마틱 데이터로부터
  모발에 가해지는 장력 상태를 "유사도 점수" 형태로 평가하는 폼스처 엔진의 핵심 개념.
  ⚠️ **금지 해석: 물리적 장력의 수치 계산(N 단위 등)이 아님. 물리 장력 계산식·
  변환 수식 작성은 원칙적으로 금지 — 키네마틱 유사도 점수만 사용.**
- **BYOD (Bring Your Own Device)**: 학습자가 자기 스마트폰 카메라로 참여하는
  저비용 침투 전략. Phase 1의 입력 소스 전제.
- **폐색 테스트 (T-1)**: 모발·가위·빗·두상이 손을 가리는 조건에서 트래킹 정확도를
  검증하는 폼스처 최우선 기술 리스크 실험.
- **공간비디오 (Spatial Video)**: Apple MV-HEVC 포맷 스테레오 영상. 마스터 모션데이터
  수집 규격 후보 (Vision Pro 기준 baseline 63.796mm). iPhone 15 Pro 이상 촬영 가능.

## 운영 용어

- **ORD-ID**: 발주 추적 식별자, 형식 ORD-YYYYMMDD-NN. orders.db(SQLite)에 영속 저장.
- **발주**: 대표가 #발주 채널에 내리는 작업 지시. 생애주기 7단계:
  접수→분해→실행중→CI심판→감리→승인대기→완결/반려. (CI심판·감리 정의는 위
  "심판 계층 정의" 절 참조 — 두 단계를 혼동하지 말 것)
- **교차 2심**: 산출물을 만든 모델과 다른 혈통의 모델이 감리하는 원칙
  (예: Claude 산출물 → GPT 감리, GPT 산출물 → Claude 감리). 위 "감리" 단계의 실제 구현.
- **CREATE / SCALE 모드**: CREATE=추론 우선 신규 설계 국면, SCALE=승인된 계획의
  효율 실행 국면.
- **바통**: 채팅 세션 간 핸드오프 문서. Google Drive theheals-바통 폴더,
  최신본 판별은 "최신 날짜+업로드 시간" 기준 (버전번호 아님).
- **INBOX 파이프라인**: 지아 산출물 → Drive theheals-inbox → 로컬 미러 →
  볼트 _inbox/ → vault-sync 자동 커밋 → 대표 승인 승격 경로.
- **볼트 (vault)**: GitHub theheals-engine-vault. 구조: 00_RAW / 10_WIKI /
  20_SKILLS / 90_SCHEMA. 로컬 경로 C:\Heals\theheals-engine-vault.
- **models.yaml**: 모델 배정의 유일한 기준 파일. **계획서·문서에 모델명 기재 시
  이 파일의 실제 값만 인용할 것 — 기억이나 일반지식으로 모델명 기재 금지.**
  (2026-07-11 모델명 날조 사고 발생)
- **PR-only 원칙**: main 브랜치 직접 커밋 전면 금지. 모든 변경은 브랜치→PR→
  CI→대표 승인→머지.
